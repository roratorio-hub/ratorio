## 0. 差分の本質（テスト方針の土台）

データファイルの大量削除（card.dat.js -2790 行 等）は `window.CARD_ID_*=...` compat ブロックの
除去のみで **データ内容は不変**。変更タイプは 4 つに集約される。

| # | 変更タイプ | 主な対象 | リスク | 「エラー無し」で十分か |
|---|---|---|---|---|
| A | **select2 → Tom Select 置換**（Phase 3d） | 装備/カード/職業/モンスターマップの全ドロップダウン | UI 挙動が別ライブラリに丸ごと変わる | ❌ 不十分 |
| B | **inline handler → addEventListener**（eventsetup.js 群） | calcx の各ボタン/チェック/セレクト + roro/other 7 ページ | 配線漏れ・誤 ID = 押しても無反応 | ❌ 不十分 |
| C | **window compat ブロック除去 + import 化** | ほぼ全ファイル（foot.js -1245, head.js 860 等） | 未 import = コードパス到達で ReferenceError | ✅ 概ね妥当 |
| D | **関数移動**（C-1 skillstate.js / C-2 CExtraInfoDataBridge.js） | パススキル・追加情報エリア | 同上 | ✅ 概ね妥当 |

### A と B は「エラーが出ない」では見落とす

- **A（Tom Select）**: 値が適用されず計算が更新されなくても **例外は出ない**。さらに
  インテグレーションテストは `dispatchEvent` を使うため select2/Tom Select の実ブラウザ挙動の
  乖離を検出できない（`reference.md` 参照）。→ **実ブラウザでの目視操作が必須**。
- **B（再配線）**: `wire()` の ID typo や呼び忘れがあると、クリックしても **静かに何も起きない**
  だけで例外は出ない。→ **「操作 → 期待した効果が出る」まで確認が必要**。
- **C/D** は「全機能を一通り操作してコンソールにエラーが出ないか」で妥当。ただし約 160 ファイルに
  分散しているので **網羅的に**叩く必要がある。
