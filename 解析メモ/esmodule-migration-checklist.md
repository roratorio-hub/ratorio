# ESModule 移行後 ブラウザ動作確認チェックリスト

対象ページ: `ro4/m/calcx.html` および `roro/other/` 配下の各ページ

---

## 前提：ESModule移行で壊れやすいポイント

このプロジェクトは `CGlobalConstManager` / `CInstanceManager` 等を始め、200+ のJSファイルが  
**グローバルスコープへの副作用**に依存している。ESModule化で以下が壊れやすい:

- `type="module"` は暗黙的に `defer` → `window.onload` / `body onload=` より後に実行される場合がある
- モジュールスコープに閉じ込められた変数は他スクリプトから `window.XXX` 参照できなくなる
- `base href="../../roro/m/"` と ESModule の相対パス解決が競合する可能性
- jQuery / Select2 / html2canvas / Chart.js の CDN 読み込み順と初期化タイミングのずれ
